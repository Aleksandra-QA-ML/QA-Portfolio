# Test Plan: E-commerce Checkout Flow (Demo Store)

**Linked Issue:** #1

## 1. Introduction
This test plan outlines the strategy for manual functional testing of the checkout process on the Demo Shopping platform. The goal is to ensure that a user can successfully navigate from the cart to order confirmation without errors.

## 2. Scope of Testing
* **Functional:** Product addition, cart updates, discount application, and final checkout.
* **UI/UX:** Form validation, error messaging, and button responsiveness.
* **Data Integrity:** Calculation of totals (subtotal, tax, shipping).

## 3. Test Strategy
* **Approach:** Manual Black-Box Testing.
* **Techniques:** Boundary Value Analysis (for item quantities), Equivalence Partitioning (for input fields).
* **Environment:** * Browser: Google Chrome (Latest)
    * Device: Desktop / Mobile (Responsive view)

## 4. Acceptance Criteria
* Users can add items to the cart and see correct totals.
* Mandatory fields in the checkout form trigger validation errors if left empty.
* Order confirmation is displayed upon successful submission.
